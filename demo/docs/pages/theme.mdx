# Theme

All components share CSS design tokens and support light and dark modes through `ThemeProvider`.

## Setup

Wrap your app once near the root:

```tsx title="App.tsx"
import { ThemeProvider } from "@rfdtech/components";
import "@rfdtech/components/style.css";

<ThemeProvider defaultTheme="system">
  <App />
</ThemeProvider>
```

Theme styles load automatically when you import from `@rfdtech/components`. You still need `@rfdtech/components/style.css` in your app entry for component CSS.

## Modes

| Value | Behavior |
|-------|----------|
| `light` | Always use the light palette |
| `dark` | Always use the dark palette |
| `system` | Follow `prefers-color-scheme` (default) |

## Toggle at runtime

```tsx title="App.tsx"
import { useTheme } from "@rfdtech/components";

function ThemeSwitcher() {
  const { theme, setTheme, resolvedTheme } = useTheme();

  return (
    <select
      value={theme}
      onChange={(event) => setTheme(event.target.value as "light" | "dark" | "system")}
    >
      <option value="light">Light</option>
      <option value="dark">Dark</option>
      <option value="system">System</option>
    </select>
  );
}
```

`resolvedTheme` is the palette currently applied (`light` or `dark`).

## CSS tokens

Tokens are defined in `theme.css` and applied on `.clet-theme` and `document.documentElement` (for portaled overlays such as modals and popovers).

Every **color** token below also has a `--clet-*` alias with the identical value (e.g. `--clet-primary` / `--clet-primary`) — **prefer `--clet-*` in new code**; `--clet-*` keeps working unchanged (each `--clet-*` color token is defined as `var(--clet-*)` under the hood, so overriding either name has the same effect). Non-color tokens (radius, shadow, z-index, font) only have a `--clet-*` name.

### Colors

| Token | Use |
|-------|-----|
| `--clet-bg` | Input, popover, and card backgrounds |
| `--clet-text` | Body text |
| `--clet-text-secondary` | Secondary text, icons |
| `--clet-text-muted` | Muted/placeholder text |
| `--clet-border` | Borders |
| `--clet-border-strong` | Hover borders, active borders |
| `--clet-hover` | Hover backgrounds |
| `--clet-surface-subtle` | Sidebar, panel backgrounds |
| `--clet-surface-dark` | Active sidebar states |
| `--clet-surface-card` | Card component background (defaults to `--clet-bg`) |
| `--clet-main-bg` | Page/app layout background |
| `--clet-overlay` | Modal/dialog backdrops |

### Semantic

| Token | Use |
|-------|-----|
| `--clet-primary` | Buttons, focus rings, accents, avatar bg |
| `--clet-primary-light` | Focus ring background, selected backgrounds |
| `--clet-secondary` | Gold brand accent (e.g. active `SidebarLink`) |
| `--clet-on-primary` | Text on primary backgrounds |
| `--clet-focus` | Focus ring color (defaults to `--clet-primary`) |
| `--clet-error` | Invalid borders, error text, destructive actions |
| `--clet-error-text` | Error message text |
| `--clet-error-bg` | Destructive action hover background |
| `--clet-success` | Success trend, success variant color |
| `--clet-warning` | Warning trend, warning variant color |

### Radius

| Token | Default | Use |
|-------|---------|-----|
| `--clet-radius-base` | `8px` | Default border radius for inputs, menus, cards |
| `--clet-radius-xl` | `16px` | Pill inputs, header search bar, popovers |
| `--clet-radius-2xl` | `24px` | Sidebar, app layout panels |

### Shadow

| Token | Use |
|-------|-----|
| `--clet-shadow-sm` | Subtle elevation |
| `--clet-shadow-md` | Medium popovers, dropdowns |
| `--clet-shadow-lg` | Large popovers, command dialog |

### Z-index

| Token | Value | Use |
|-------|-------|-----|
| `--clet-z-popover` | `1000` | Popovers, dropdowns |
| `--clet-z-overlay` | `1100` | Modal/dialog overlays |
| `--clet-z-modal` | `1101` | Modal content |
| `--clet-z-select` | `1200` | Radix select popovers |
| `--clet-z-alert` | `1200` | Alert dialogs |
| `--clet-z-alert-content` | `1201` | Alert dialog content |
| `--clet-z-dropdown` | `1300` | Dropdown menus (highest) |

### Other

| Token | Use |
|-------|-----|
| `--clet-font` | Font family stack |

Override any `--clet-*` CSS variable with the `cletTheme()` runtime helper, or
by hand with plain CSS.

### `cletTheme()` (recommended)

Call once at startup, after importing the library CSS. It injects a
`<style>` tag with the light/dark overrides — no CSS file, no build
config:

```ts title="src/main.tsx"
import "@rfdtech/components/style.css";
import { cletTheme } from "@rfdtech/components";

cletTheme({
  all: {
    primary: "#1d4ed8", // used by both modes unless a mode below overrides it
  },
  light: {
    primaryLight: "#eff6ff",
    focus: "#1d4ed8",
    onPrimary: "#ffffff",
  },
  dark: {
    primary: "#ef4444", // wins over "all" for dark specifically
    primaryLight: "#3f1515",
    focus: "#ef4444",
    onPrimary: "#ffffff",
  },
});
```

Keys are the **camelCase** form of the `--clet-*` token names documented
above (`--clet-primary` → `primary`, `--clet-radius-base` → `radiusBase`).
`all`, `light`, and `dark` are all optional:

- `all` applies to both modes, **per token** — set a token in `all` and
  it's used everywhere unless that specific mode also sets it.
- A token set in `light`/`dark` always wins over the same token in `all`.
- Omit a token everywhere to leave the library's own default untouched.

Calling `cletTheme()` again (e.g. on hot reload) replaces the previous
overrides rather than stacking a new `<style>` tag.

#### Recommended file: `src/clet.theme.ts`

There's no build-time auto-discovery for a theme file (no Vite plugin scans
for one, unlike e.g. `tailwind.config.ts`) — `cletTheme()` is a plain runtime
function, so it has to be explicitly imported once. To keep every consumer's
override in the same predictable place, put the `cletTheme(...)` call in its
own file named **`src/clet.theme.ts`** and import it for its side effect,
right after the library CSS, in your entry point:

```ts title="src/clet.theme.ts"
import { cletTheme } from "@rfdtech/components";

cletTheme({
  all: { primary: "#1d4ed8" },
  dark: { primary: "#ef4444", primaryLight: "#3f1515" },
});
```

```ts title="src/main.tsx"
import "@rfdtech/components/style.css";
import "./clet.theme"; // side-effect import — runs cletTheme() once at startup
import { ThemeProvider } from "@rfdtech/components";
```

One file, one canonical name, imported once — not a hard requirement (any
file/location works, since it's just a function call), but the convention
every CLET project should follow so a theme override is always easy to find.

#### Type safety

`cletTheme()` is fully typed against the library's real token set — both
the **key** and the **value**:

- Unknown or mistyped keys are a type error (`primry` won't compile, and
  neither will the raw CSS string `"--clet-primary"` — keys are camelCase).
- Each token only accepts values shaped like its CSS group:

| Group | Accepts | Example tokens |
|-------|---------|-----------------|
| Color | hex, `rgb()`/`rgba()`, `hsl()`/`hsla()`, `oklch()`, or `var(--x)` | `primary`, `bg`, `focus` |
| Length | `px`/`rem`/`em`/`%`, `"0"`, or `var(--x)` | `radiusBase`, `gap` |
| Z-index | `number` | `zModal`, `zPopover` |
| Opacity | `number` or a `%` string | tokens matching `*opacity*` |
| Shadow / font / misc | any `string` (shadow and font syntax is too varied to constrain further) | `shadowMd`, `font` |

```ts
cletTheme({
  light: {
    zHeader: 20, // ✅ number
    // zHeader: "20", ❌ type error — z-index tokens are numbers
    // primary: "blue", ❌ type error — not a hex/rgb/hsl/oklch/var()
  },
});
```

This catches the most common mistakes — misspelled/wrong-cased token
names and mismatched value shapes — without trying to fully validate CSS
grammar (e.g. it won't catch `rgb(999 999 999)`, out-of-range channel
values).

#### Component-scoped overrides

Some components expose their own local tokens (e.g. `AppHeader`'s
`--clet-app-header-bg`, exposed here as `bg`) in addition to the global
ones above. Target a component by name under `components`; only that
component's own tokens are valid there, and each also supports
`all`/`light`/`dark`:

```ts
cletTheme({
  components: {
    AppHeader: {
      all: { font: "Inter, sans-serif" },
      light: { bg: "#111827", color: "#ffffff" },
      dark: { bg: "#000000" },
    },
    Card: {
      light: { padding: "24px" },
    },
  },
});
```

`components` keys are the exported component names (`AppHeader`,
`Card`, `MetricCard`, ...). Passing `padding` (Card's token) under
`AppHeader` is a type error — each component only accepts the tokens it
actually declares.

### Manual CSS

You can also override tokens by hand by declaring new values on the
same selectors the library uses.

**Step 1 — Import the library CSS** so its defaults are loaded:

```ts title="src/main.tsx"
import "@rfdtech/components/style.css";
```

**Step 2 — Create a theme override file** and import it AFTER the
library CSS. CSS source order matters: later rules with the same
specificity win.

```ts title="src/main.tsx"
import "@rfdtech/components/style.css";
import "./styles/theme.css"; // imported after
```

**Step 3 — Override the tokens.** The library writes its defaults on
these selectors:

| Selector | When it matches |
|----------|-----------------|
| `:root` | No theme attribute is set (no `ThemeProvider` mounted) |
| `:root[data-clet-theme="light"]` | Light mode on `<html>` — `ThemeProvider` writes the attribute here |
| `:root[data-clet-theme="dark"]` | Dark mode on `<html>` |
| `.clet-theme` | A consumer wrapper `<div class="clet-theme">` |
| `.clet-theme[data-clet-theme="light"]` | Wrapper + light attribute |
| `.clet-theme[data-clet-theme="dark"]` | Wrapper + dark attribute |

A bare `:root { ... }` only matches the first row, so the library's
default keeps winning on rows 2–6 once `ThemeProvider` mounts. To
cover all of them, list every selector in one rule and hardcode the
values (don't chain `var(--something)` — just write the color):

```css title="src/styles/theme.css"
:root,
:root[data-clet-theme="light"],
.clet-theme[data-clet-theme="light"],
:root[data-clet-theme="dark"],
.clet-theme[data-clet-theme="dark"] {
  --clet-primary: #1d4ed8;
  --clet-primary-light: #eff6ff;
  --clet-focus: #1d4ed8;
  --clet-on-primary: #ffffff;
}
```

That one block overrides the primary color in light mode, dark mode,
and the unthemed fallback. The same shape works for any other color
token — list every selector, hardcode the value, using its `--clet-*`
name (or the equivalent `--clet-*` name — both work). Non-color tokens
only have a `--clet-*` name.

## Tailwind v4 integration

All `--clet-*` tokens are registered with Tailwind v4's token system via a `@theme` block shipped in `style.css`. No extra configuration needed.

### Consumer setup

```css title="app/src/index.css"
@import "tailwindcss";
@import "@rfdtech/components/style.css";
```

The consumer's Tailwind build processes the `@theme` block and generates utility classes:

| Utility | Resolves to |
|---------|-------------|
| `bg-primary` / `text-primary` | `var(--clet-primary)` |
| `bg-primary-foreground` / `text-primary-foreground` | `var(--clet-on-primary)` |
| `bg-background` / `text-foreground` | `var(--clet-main-bg)` / `var(--clet-text)` |
| `bg-secondary` / `text-secondary-foreground` | `var(--clet-surface-dark)` / `var(--clet-text-secondary)` |
| `bg-muted` / `text-muted-foreground` | `var(--clet-surface-subtle)` / `var(--clet-text-secondary)` |
| `bg-accent` / `text-accent-foreground` | `var(--clet-hover)` / `var(--clet-text)` |
| `bg-destructive` / `text-destructive-foreground` | `var(--clet-error)` / `var(--clet-error-text)` |
| `bg-card` / `text-card-foreground` | `var(--clet-bg)` / `var(--clet-text)` |
| `bg-popover` / `text-popover-foreground` | `var(--clet-bg)` / `var(--clet-text)` |
| `border` / `input` / `ring` | `var(--clet-border)` / `var(--clet-border-strong)` / `var(--clet-focus)` |
| `bg-success` / `bg-warning` | `var(--clet-success)` / `var(--clet-warning)` |
| `rounded-lg` | `var(--clet-radius-base)` |
| `shadow-sm` / `shadow-md` / `shadow-lg` | `var(--clet-shadow-sm)` / etc. |
| `font-sans` | `var(--clet-font)` |

### Dark mode

Both conventions are supported:

| Selector | Convention |
|----------|-----------|
| `<html class="dark">` | Tailwind v4 default |
| `<html data-clet-theme="dark">` | CLET native (`ThemeProvider`) |

### Non-Tailwind consumers

No impact. Browsers silently ignore `@theme` as an unknown at-rule. CLET components continue to reference `--clet-*` tokens directly.

## Controlled mode

```tsx title="App.tsx"
const [theme, setTheme] = useState<"light" | "dark" | "system">("system");

<ThemeProvider theme={theme} onThemeChange={setTheme}>
  <App />
</ThemeProvider>
```

When `theme` is passed, the provider is controlled and `setTheme` from `useTheme()` calls `onThemeChange` only.
