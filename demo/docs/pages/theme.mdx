# Theme

All components share CSS design tokens and support light and dark modes through `ThemeProvider`.

## Setup

Wrap your app once near the root:

```tsx title="App.tsx"
import { ThemeProvider } from "@rfdtech/components";
import "@rfdtech/components/style.css";

<ThemeProvider defaultTheme="system">
  <App />
</ThemeProvider>
```

Theme styles load automatically when you import from `@rfdtech/components`. You still need `@rfdtech/components/style.css` in your app entry for component CSS.

## Modes

| Value | Behavior |
|-------|----------|
| `light` | Always use the light palette |
| `dark` | Always use the dark palette |
| `system` | Follow `prefers-color-scheme` (default) |

## Toggle at runtime

```tsx title="App.tsx"
import { useTheme } from "@rfdtech/components";

function ThemeSwitcher() {
  const { theme, setTheme, resolvedTheme } = useTheme();

  return (
    <select
      value={theme}
      onChange={(event) => setTheme(event.target.value as "light" | "dark" | "system")}
    >
      <option value="light">Light</option>
      <option value="dark">Dark</option>
      <option value="system">System</option>
    </select>
  );
}
```

`resolvedTheme` is the palette currently applied (`light` or `dark`).

## CSS tokens

Tokens are defined in `theme.css` and applied on `.gsl-theme` and `document.documentElement` (for portaled overlays such as modals and popovers).

### Colors

| Token | Use |
|-------|-----|
| `--gsl-bg` | Input, popover, and card backgrounds |
| `--gsl-text` | Body text |
| `--gsl-text-secondary` | Secondary text, icons |
| `--gsl-text-muted` | Muted/placeholder text |
| `--gsl-border` | Borders |
| `--gsl-border-strong` | Hover borders, active borders |
| `--gsl-hover` | Hover backgrounds |
| `--gsl-surface-subtle` | Sidebar, panel backgrounds |
| `--gsl-surface-dark` | Active sidebar states |
| `--gsl-surface-card` | Card component background (defaults to `--gsl-bg`) |
| `--gsl-page-bg` | Page/app layout background |
| `--gsl-overlay` | Modal/dialog backdrops |

### Semantic

| Token | Use |
|-------|-----|
| `--gsl-primary` | Buttons, focus rings, accents, avatar bg |
| `--gsl-primary-light` | Focus ring background, selected backgrounds |
| `--gsl-on-primary` | Text on primary backgrounds |
| `--gsl-focus` | Focus ring color (defaults to `--gsl-primary`) |
| `--gsl-error` | Invalid borders, error text, destructive actions |
| `--gsl-error-text` | Error message text |
| `--gsl-error-bg` | Destructive action hover background |
| `--gsl-success` | Success trend, success variant color |
| `--gsl-warning` | Warning trend, warning variant color |

### Radius

| Token | Default | Use |
|-------|---------|-----|
| `--gsl-radius-base` | `8px` | Default border radius for inputs, menus, cards |
| `--gsl-radius-xl` | `16px` | Pill inputs, header search bar, popovers |
| `--gsl-radius-2xl` | `24px` | Sidebar, app layout panels |

### Shadow

| Token | Use |
|-------|-----|
| `--gsl-shadow-sm` | Subtle elevation |
| `--gsl-shadow-md` | Medium popovers, dropdowns |
| `--gsl-shadow-lg` | Large popovers, command dialog |

### Z-index

| Token | Value | Use |
|-------|-------|-----|
| `--gsl-z-popover` | `1000` | Popovers, dropdowns |
| `--gsl-z-overlay` | `1100` | Modal/dialog overlays |
| `--gsl-z-modal` | `1101` | Modal content |
| `--gsl-z-select` | `1200` | Radix select popovers |
| `--gsl-z-alert` | `1200` | Alert dialogs |
| `--gsl-z-alert-content` | `1201` | Alert dialog content |
| `--gsl-z-dropdown` | `1300` | Dropdown menus (highest) |

### Other

| Token | Use |
|-------|-----|
| `--gsl-font` | Font family stack |

Override any `--gsl-*` CSS variable by declaring a new value on the
same selectors the library uses.

**Step 1 — Import the library CSS** so its defaults are loaded:

```ts title="src/main.tsx"
import "@rfdtech/components/style.css";
```

**Step 2 — Create a theme override file** and import it AFTER the
library CSS. CSS source order matters: later rules with the same
specificity win.

```ts title="src/main.tsx"
import "@rfdtech/components/style.css";
import "./styles/theme.css"; // imported after
```

**Step 3 — Override the tokens.** The library writes its defaults on
these selectors:

| Selector | When it matches |
|----------|-----------------|
| `:root` | No theme attribute is set (no `ThemeProvider` mounted) |
| `:root[data-gsl-theme="light"]` | Light mode on `<html>` — `ThemeProvider` writes the attribute here |
| `:root[data-gsl-theme="dark"]` | Dark mode on `<html>` |
| `.gsl-theme` | A consumer wrapper `<div class="gsl-theme">` |
| `.gsl-theme[data-gsl-theme="light"]` | Wrapper + light attribute |
| `.gsl-theme[data-gsl-theme="dark"]` | Wrapper + dark attribute |

A bare `:root { ... }` only matches the first row, so the library's
default keeps winning on rows 2–6 once `ThemeProvider` mounts. To
cover all of them, list every selector in one rule and hardcode the
values (don't chain `var(--something)` — just write the color):

```css title="src/styles/theme.css"
:root,
:root[data-gsl-theme="light"],
.gsl-theme[data-gsl-theme="light"],
:root[data-gsl-theme="dark"],
.gsl-theme[data-gsl-theme="dark"] {
  --gsl-primary: #1d4ed8;
  --gsl-primary-light: #eff6ff;
  --gsl-focus: #1d4ed8;
  --gsl-on-primary: #ffffff;
}
```

That one block overrides the primary color in light mode, dark mode,
and the unthemed fallback. The same shape works for any other
`--gsl-*` token — list every selector, hardcode the value.

## Tailwind v4 integration

All `--gsl-*` tokens are registered with Tailwind v4's token system via a `@theme` block shipped in `style.css`. No extra configuration needed.

### Consumer setup

```css title="app/src/index.css"
@import "tailwindcss";
@import "@rfdtech/components/style.css";
```

The consumer's Tailwind build processes the `@theme` block and generates utility classes:

| Utility | Resolves to |
|---------|-------------|
| `bg-primary` / `text-primary` | `var(--gsl-primary)` |
| `bg-primary-foreground` / `text-primary-foreground` | `var(--gsl-on-primary)` |
| `bg-background` / `text-foreground` | `var(--gsl-page-bg)` / `var(--gsl-text)` |
| `bg-secondary` / `text-secondary-foreground` | `var(--gsl-surface-dark)` / `var(--gsl-text-secondary)` |
| `bg-muted` / `text-muted-foreground` | `var(--gsl-surface-subtle)` / `var(--gsl-text-secondary)` |
| `bg-accent` / `text-accent-foreground` | `var(--gsl-hover)` / `var(--gsl-text)` |
| `bg-destructive` / `text-destructive-foreground` | `var(--gsl-error)` / `var(--gsl-error-text)` |
| `bg-card` / `text-card-foreground` | `var(--gsl-bg)` / `var(--gsl-text)` |
| `bg-popover` / `text-popover-foreground` | `var(--gsl-bg)` / `var(--gsl-text)` |
| `border` / `input` / `ring` | `var(--gsl-border)` / `var(--gsl-border-strong)` / `var(--gsl-focus)` |
| `bg-success` / `bg-warning` | `var(--gsl-success)` / `var(--gsl-warning)` |
| `rounded-lg` | `var(--gsl-radius-base)` |
| `shadow-sm` / `shadow-md` / `shadow-lg` | `var(--gsl-shadow-sm)` / etc. |
| `font-sans` | `var(--gsl-font)` |

### Dark mode

Both conventions are supported:

| Selector | Convention |
|----------|-----------|
| `<html class="dark">` | Tailwind v4 default |
| `<html data-gsl-theme="dark">` | GSL native (`ThemeProvider`) |

### Non-Tailwind consumers

No impact. Browsers silently ignore `@theme` as an unknown at-rule. GSL components continue to reference `--gsl-*` tokens directly.

## Controlled mode

```tsx title="App.tsx"
const [theme, setTheme] = useState<"light" | "dark" | "system">("system");

<ThemeProvider theme={theme} onThemeChange={setTheme}>
  <App />
</ThemeProvider>
```

When `theme` is passed, the provider is controlled and `setTheme` from `useTheme()` calls `onThemeChange` only.
