# Dialog

Compound dialog primitives for modal overlays such as forms, detail panels, and settings. Wraps `@radix-ui/react-dialog`; styled parts apply default CLET surface styles.

## Example

<ExampleTabs
  title="Edit profile"
  preview={<DialogPreview />}
  code={dialogSource}
/>

Click **Edit profile** to open the dialog with `?dialog=edit-profile&dialog.userId=42`. Close the dialog or use the browser back button to clear the params.

## URL state

Sync open state with URL search params using [`useDialogSearchParam`](/docs/hooks). The example above wires `open` and `onOpenChange` from the hook and opens via `openWith({ userId: "42" })`.

| URL | Meaning |
|-----|---------|
| `?dialog=edit-profile` | Dialog `edit-profile` is open |
| `?dialog=edit-profile&dialog.userId=42` | Same dialog with `userId` context |

Use a `Button` (or link) with `openWith()` instead of `DialogTrigger` when open state is URL-controlled — Radix triggers manage local state and conflict with search-param sync. Pass optional scalar `data` fields through `openWith`; parsed values are available on `data` as strings.

For React Router or other routers, pass a custom `adapter` — see [Hooks](/docs/hooks#react-router).

## Components

### `Dialog`

Root wrapper. Accepts Radix `Dialog.Root` props such as `open`, `defaultOpen`, and `onOpenChange`.

### `DialogTrigger`

Wraps the element that opens the dialog. Use `asChild` to merge props onto a child such as `Button`.

### `DialogPortal`

Renders overlay and content in a portal. Wrap `DialogOverlay` and `DialogContent` inside the portal.

### `DialogOverlay`

Styled backdrop behind the dialog panel.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `DialogOverlayClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the overlay element |

Also accepts remaining Radix `Dialog.Overlay` props.

### `DialogContent`

Styled centered modal panel.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `showCloseButton` | `boolean` | `false` | Shows a top-right close control with an accessible label |
| `classNames` | `DialogContentClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the content element |

Also accepts remaining Radix `Dialog.Content` props.

### `DialogTitle`

Accessible dialog heading. Render inside `DialogContent`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `DialogTitleClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the title element |

### `DialogDescription`

Optional supporting text below the title.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `DialogDescriptionClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the description element |

### Other exports

| Export | Radix primitive | Notes |
|--------|-----------------|-------|
| `DialogClose` | `Close` | Closes the dialog when activated; used by the built-in close button and custom actions |

## Types

### `DialogOverlayProps`

Extends Radix `DialogOverlayProps` with optional `classNames`.

### `DialogContentProps`

Extends Radix `DialogContentProps` with `showCloseButton` and optional `classNames`.

### `DialogTitleProps` / `DialogDescriptionProps`

Extend Radix title and description props with optional `classNames`.

### `DialogOverlayClassNames`

| Key | Applied to |
|-----|------------|
| `overlay` | Overlay element (`.clet-dialog__overlay`) |

### `DialogContentClassNames`

| Key | Applied to |
|-----|------------|
| `content` | Content element (`.clet-dialog__content`) |
| `close` | Built-in close button (`.clet-dialog__close`) |

### `DialogTitleClassNames`

| Key | Applied to |
|-----|------------|
| `title` | Title element (`.clet-dialog__title`) |

### `DialogDescriptionClassNames`

| Key | Applied to |
|-----|------------|
| `description` | Description element (`.clet-dialog__description`) |

## Notes

- Portaled content inherits the active theme via `data-clet-theme` on `document.documentElement` when using `ThemeProvider`.
- **URL-driven open state** is the recommended pattern: use `useDialogSearchParam` with `open`, `onOpenChange`, and `openWith` (see [URL state](#url-state) above). For local-only dialogs, `DialogTrigger asChild` keeps focus and keyboard behavior on your button or link.
- The panel is centered with `width: min(480px, calc(100vw - 32px))` and scrollable content capped at `min(85vh, calc(100dvh - 32px))` so it fits mobile viewports.
- The built-in close control uses a 40×40px hit area for touch-friendly dismissal.
- Enter/exit animations are disabled when `prefers-reduced-motion: reduce` is set.
- Base `clet-dialog*` classes are always applied; `classNames` and `className` append consumer classes for Tailwind utilities.

## Do

- For a custom leading icon next to the title, wrap both in `<div className="flex items-center gap-2">` with the icon at `size-5 shrink-0`.

## Don't

- Override `DialogContent` padding via `classNames={{content: "p-0 overflow-hidden"}}` — it manages its own padding.
- Wrap Title/Description/actions in extra padded/gapped wrapper divs — they sit directly inside `DialogContent`, causing doubled gaps.
- Add a separate footer component — the actions row is just `<div className="flex justify-end gap-2">` directly inside `DialogContent`.
