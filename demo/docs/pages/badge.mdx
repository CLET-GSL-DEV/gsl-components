# Badge

Compact inline label for status, counts, and metadata. Use semantic variants for success, warning, and error states, or `outline` for a bordered neutral badge.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `BadgeVariant` | `"default"` | Visual style |
| `size` | `BadgeSize` | `"sm"` | Font size and padding |
| `classNames` | `BadgeClassNames` | ‚Äî | Part-level class overrides |
| `className` | `string` | ‚Äî | Merged onto the root (same as `classNames.root`) |
| `children` | `ReactNode` | ‚Äî | Badge label |

Also accepts standard native `span` attributes (`id`, `aria-*`, etc.).

## Types

### `BadgeProps`

Props for `Badge`. Extends `HTMLAttributes<HTMLSpanElement>` with `variant`, `size`, and `classNames`.

### `BadgeClassNames`

Part-level class names merged after base `gsl-badge*` styles:

| Key | Applied to |
|-----|------------|
| `root` | Outer `<span>` element |

### `BadgeVariant`

`"default" | "primary" | "success" | "warning" | "error" | "outline"`

### `BadgeSize`

`"sm" | "md"`

## Notes

- Badges are display-only; they do not include interactive hover or focus styles.
- Base `gsl-*` classes are always applied; `classNames` and `className` append consumer classes for Tailwind utilities.
- Long labels truncate with an ellipsis when space is constrained (`max-width: 100%`).
