# PhoneNumberInput

Phone number input with a country code selector (flag + dial code prefix). Built on `@radix-ui/react-popover` with GSL styling. Auto-detects the country from the number prefix as the user types. Supports controlled and uncontrolled value.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | Full phone number value (e.g. `"+2332054321022"`) |
| `onChange` | `(value: string) => void` | — | Change handler |
| `placeholder` | `string` | `"Enter phone number"` | Input placeholder |
| `defaultCountry` | `string` | — | Default country ISO code (overrides auto-detection) |
| `invalid` | `boolean` | `false` | Show error border styling |
| `disabled` | `boolean` | `false` | Disable interaction |
| `name` | `string` | — | Form field name (for react-hook-form) |
| `classNames` | `PhoneNumberInputClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root element |

Also accepts remaining `HTMLAttributes<HTMLDivElement>`, excluding `onChange`, `value`, and `defaultValue`.

## Usage with react-hook-form

```tsx
import { useForm } from "react-hook-form";
import { Form, FormField, Field, FieldControl, FieldLabel, FieldError, PhoneNumberInput } from "@rfdtech/components";

function PhoneForm() {
  const form = useForm<{ phone: string }>({ defaultValues: { phone: "" } });

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit((d) => console.log(d.phone))}>
        <FormField
          control={form.control}
          name="phone"
          rules={{ required: "Phone number is required" }}
          render={({ field, fieldState }) => (
            <Field invalid={!!fieldState.error}>
              <FieldLabel>Phone number</FieldLabel>
              <FieldControl>
                <PhoneNumberInput value={field.value} onChange={field.onChange} />
              </FieldControl>
              <FieldError>{fieldState.error?.message}</FieldError>
            </Field>
          )}
        />
      </form>
    </Form>
  );
}
```

## Notes

- Country auto-detection: as the user types, the component matches the prefix against dial codes and updates the selected country. The sorted check uses longest dial code first to avoid partial matches (e.g., `+233` matches Ghana, not `+23`).
- Countries are from the `countries` utility — same dataset used by `CountrySelector`.
- The value includes the full number with the dial code prefix (e.g. `+2332054321022`). The `+` prefix is always included.
- Changing the country dropdown updates the dial code prefix in the input value.
