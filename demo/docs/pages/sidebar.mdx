# Sidebar

Compound sidebar primitives for app shells and section navigation. Desktop shows a sticky card-style rail with optional collapse; mobile uses an offcanvas drawer with a toggle trigger and dismissible overlay.

  code={sidebarSource}
/>

## Variants

`Sidebar` accepts a `variant` prop that controls its surface style.

- **`"default"`** (default) — card panel surface (`--gsl-surface-subtle` background, rounded corners).
- **`"plain"`** — transparent background with a right border instead of the card panel, and no border radius. Used for dashboard shells where the sidebar should sit flush against the layout edge.

<ExampleTabs
  title='variant="plain" — transparent background with a right border'

## Loading state

Set `loading` on `SidebarLink` to shimmer an icon+label placeholder row instead of the real link — useful while nav items are still being fetched. The placeholder is non-interactive and ignores `icon`/`children`/`to`/`onClick`/`active`.

## React Router

Use `asChild` on `SidebarLink` to merge styles onto `NavLink`:

```tsx
<SidebarLink asChild active={isActive}>
  <NavLink to="/dashboard">Dashboard</NavLink>
</SidebarLink>
```

## Components

### `SidebarProvider`

Context wrapper for open state, collapsed state, and mobile detection.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | — | Controlled open state (mobile drawer) |
| `defaultOpen` | `boolean` | `false` | Initial open state (uncontrolled) |
| `onOpenChange` | `(open: boolean) => void` | — | Called when open state changes |
| `collapsed` | `boolean` | — | Controlled collapsed state (desktop) |
| `defaultCollapsed` | `boolean` | `false` | Initial collapsed state (uncontrolled) |
| `onCollapsedChange` | `(collapsed: boolean) => void` | — | Called when collapsed state changes |
| `breakpoint` | `number` | `768` | Max width in px treated as mobile |
| `classNames` | `SidebarProviderClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the provider wrapper |
| `children` | `ReactNode` | — | Trigger, overlay, sidebar, and page content |

Also exports `useSidebar()` for programmatic open/close and collapse.

### `Sidebar`

Root `<aside>`. Sticky card panel on desktop; fixed offcanvas panel on mobile when open.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "plain"` | `"default"` | Visual style — see [Variants](#variants) |
| `classNames` | `SidebarClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the aside |
| `children` | `ReactNode` | — | Header, content, footer |

### `SidebarCollapse`

Desktop collapse toggle. Hidden on mobile. Sets `aria-expanded` and `aria-controls`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `SidebarCollapseClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the button |

Also accepts standard `button` attributes.

### `SidebarTrigger`

Mobile menu button. Hidden on desktop. Sets `aria-expanded` and `aria-controls`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `SidebarTriggerClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the button |
| `children` | `ReactNode` | — | Button label |

Also accepts standard `button` attributes.

### `SidebarOverlay`

Mobile backdrop. Click closes the drawer. Not rendered on desktop.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `SidebarOverlayClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the overlay button |

### `SidebarHeader`

Top branding or metadata slot. Use `gsl-sidebar__header-brand` and `gsl-sidebar__header-title` for logo + title layout alongside `SidebarCollapse`.

### `SidebarContent`

Scrollable middle section for navigation.

### `SidebarFooter`

Bottom slot separated by a top border. A plain wrapper — typically holds a [`ProfilePopover`](/profile-popover) for the signed-in user's trigger row and menu.

```tsx
<SidebarFooter>
  <ProfilePopover
    fullName="Kwame Asante"
    email="kwame@gsl.edu.gh"
    items={[
      { icon: <User size={20} />, label: "My Profile", onClick: () => navigate("/profile") },
      { icon: <Settings size={20} />, label: "Account Settings", onClick: () => navigate("/settings") },
    ]}
    onSignOut={handleSignOut}
  >
    <RoleSelect
      roles={roles}
      selectedRole={selectedRole}
      onClickRole={(role) => setSelectedRole(role.id)}
    />
  </ProfilePopover>
</SidebarFooter>
```

Navigation landmark wrapper (`<nav>`).

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `aria-label` | `string` | `"Sidebar"` | Accessible name for the nav |
| `classNames` | `SidebarNavClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the nav |

### `SidebarGroup`

Section wrapper for grouped links. Set `collapsible` to make the first `SidebarGroupLabel` child render as a button that toggles the group's content with an animated open/close.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `collapsible` | `boolean` | `false` | Render the label as a toggle button that animates content open/closed |
| `defaultExpanded` | `boolean` | `true` | Initial expanded state (uncontrolled) |
| `expanded` | `boolean` | — | Controlled expanded state |
| `onExpandedChange` | `(expanded: boolean) => void` | — | Called when expanded state changes |
| `classNames` | `SidebarGroupClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the group wrapper |
| `children` | `ReactNode` | — | Optional `SidebarGroupLabel` + nav items |

### `SidebarGroupLabel`

Uppercase section title. Renders as a `<p>` by default, or as a `<button>` toggle when the parent `SidebarGroup` has `collapsible`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `SidebarGroupLabelClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the label |
| `children` | `ReactNode` | — | Label text |

```tsx
<SidebarGroup collapsible defaultExpanded={false}>
  <SidebarGroupLabel>Advanced</SidebarGroupLabel>
  <SidebarItem>
    <SidebarLink>API Keys</SidebarLink>
  </SidebarItem>
</SidebarGroup>
```

### `SidebarItem`

Row wrapper for a single link.

### `SidebarBadge`

Trailing count or label pill inside `SidebarLink`. Must be a direct child of `SidebarLink`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `classNames` | `SidebarBadgeClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the badge |
| `children` | `ReactNode` | — | Badge text (e.g. `"New"`, `12`) |

```tsx
<SidebarLink href="/notifications" icon={<Bell size={20} />}>
  Notification Templates
  <SidebarBadge>New</SidebarBadge>
</SidebarLink>
```

### `SidebarLink`

Styled navigation link.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `active` | `boolean` | `false` | Applies active link styling |
| `asChild` | `boolean` | `false` | Merge props onto the child element |
| `icon` | `ReactNode` | — | Optional leading icon; label wraps in `.gsl-sidebar__link-label` for collapse |
| `loading` | `boolean` | `false` | Render a shimmering skeleton placeholder instead of the icon/label — see [Loading state](#loading-state) |
| `loadingLabel` | `string` | `"Loading"` | Accessible label announced while loading |
| `classNames` | `SidebarLinkClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the link |
| `href` | `string` | — | Link destination |

Also accepts standard anchor attributes when not using `asChild`. When using `asChild`, supply your own icon/label structure; collapse label hiding works best with the `icon` prop pattern. `SidebarBadge` children are not supported with `asChild`.

## Notes

- Desktop renders a **card panel** (`--gsl-bg` background, 24px radius, icon + label rows) by default. Active links use a neutral gray pill, not primary color.
- `variant="plain"` swaps the card panel for a transparent background with a right border (`--gsl-border`) and no radius — use it alongside `AppHeader`'s `"plain"` variant for a flush dashboard shell.
- Use `SidebarCollapse` in the header to collapse to a **72px icon rail** on desktop; group labels, link labels, badges, and footer content hide automatically.
- `SidebarGroup` with `collapsible` renders its first `SidebarGroupLabel` as a button and animates the content open/close with an opacity transition. The label looks identical to the static label when not collapsible — same font, size, color, and padding. The content area stays in the layout when folded (items fade to invisible but the space is reserved) so toggling doesn't shift items below it. Set `defaultExpanded` for the initial state or use `expanded` + `onExpandedChange` for controlled state. The animation respects `prefers-reduced-motion`.
- Mobile breakpoint defaults to **768px**; override with `breakpoint` on `SidebarProvider` if needed.
- On mobile, the drawer slides in from the left; overlay click and resize back to desktop close the drawer.
- Link touch targets are at least **40px** tall on mobile.
- Slide transitions are disabled when `prefers-reduced-motion: reduce` is set.
- `SidebarBadge` renders a trailing pill on the right of a link row; badges hide automatically when the sidebar is collapsed.
- When collapsed, each `SidebarLink` automatically shows a `Tooltip` (positioned to the right) with the link's label text on hover. Links with no label text skip the tooltip.
- Base `gsl-*` classes are always applied; `classNames` and `className` append consumer classes for Tailwind utilities.
