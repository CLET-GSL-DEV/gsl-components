# AppLayout

Application layout container that auto-positions `AppHeader`, `AppSidebar`, and `AppBody` by component type. Wraps `SidebarProvider` and `BreadcrumbProvider` internally. Breadcrumbs render from `useBreadcrumbs()` context, no `AppBreadcrumb` slot needed.

## Variants

`AppLayout` accepts a `variant` prop that controls how the header, sidebar, and body are arranged.

- **`"default"`** (default) — the sidebar spans the full height on the left, and the header sits above the content only.
- **`"stacked"`** — the header spans the full width on top, with the sidebar and content side by side below it. No extra styling.

The example below is the dashboard shell combination: `variant="stacked"` on `AppLayout` paired with [`AppHeader`'s `"plain"` variant](/docs/app-header#variants) and [`Sidebar`'s `"plain"` variant](/docs/sidebar#variants), plus collapsible nav groups. Use this combo when the header and sidebar should read as one solid dashboard frame rather than floating panels.

## Components

### `AppLayout`

Root layout container. Scans direct children for `componentId` matching `"AppHeader"`, `"AppSidebar"`, or `"AppBody"` and positions them automatically. Breadcrumbs are rendered between the header and body when `useBreadcrumbs()` is called from any child. `SidebarProvider` and `BreadcrumbProvider` are wrapped internally, no need to add them manually.

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Direct children — `<AppHeader>`, `<AppSidebar>`, `<AppBody>` |
| `className` | `string` | Merged onto the root element |
| `variant` | `"default" \| "stacked"` | Layout arrangement. `"default"`: sidebar full-height on the left, header above content only. `"stacked"`: full-width header on top, sidebar + content side by side below. Defaults to `"default"`. |

### `AppSidebar`

Wrapper for sidebar content. Must be a direct child of `AppLayout`. Renders inside a flex column with `max-content` width. Typically contains `<Sidebar>`.

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Sidebar content — typically `<Sidebar>` |
| `className` | `string` | Merged onto the sidebar wrapper |

`AppSidebar` renders no extra DOM element — it passes children through. The sidebar wrapper div is created by `AppLayout`.

### `AppBody`

Wrapper for main page content. Must be a direct child of `AppLayout`. Renders as `<main>` inside the content area.

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Page content — includes `<AppHeader>` if desired, plus body content |
| `className` | `string` | Merged onto the `<main>` element |

> Note: `AppHeader` renders above `AppBody` when both are direct children of `AppLayout`. If `AppHeader` is placed inside `AppBody`, it renders inline within the content area without the sticky header wrapper.

## Breadcrumbs

Breadcrumbs are rendered automatically by `AppLayout` when items are set via `useBreadcrumbs()`. No `<AppBreadcrumb>` wrapper or `<BreadcrumbProvider>` needed.

```tsx
import { useBreadcrumbs } from "@rfdtech/components";

function Page() {
  useBreadcrumbs([
    { label: "Users", href: "/users" },
    { label: "Current User" },
  ]);
  return <div>Page content</div>;
}
```

When `items` is empty, the breadcrumb section is hidden entirely.

## Design tokens

| Token | Fallback | Used for |
|-------|----------|----------|
| `--gsl-app-layout-body-gap` | `1rem` | Gap between layout sections |
| `--gsl-app-layout-gap` | `1rem` | Outer padding and flex gap |
| `--gsl-panel-bg` | `--gsl-surface-subtle` | Content area background |
| `--gsl-panel-radius` | `--gsl-radius-2xl` | Content area and body border radius |
| `--gsl-app-layout-font` | `--gsl-font-body` | Font family for the root layout element |
| `--gsl-scroll-fade` | `1` | Opacity multiplier for the top/bottom scroll fade overlays on the body |
| `--gsl-scroll-fade-height` | `calc(var(--gsl-app-layout-body-gap) * 3)` | Height of the bottom (and, unless overridden, top) scroll fade overlay |
| `--gsl-scroll-fade-width` | `100%` | Width of the top/bottom scroll fade overlays |

## Notes

- The header is sticky (`position: sticky; top: 0`), so it stays visible when the body scrolls.
- `AppLayout` sets `height: 100vh; overflow: hidden` on itself and `overflow-y: auto` on the body area — the page scrolls within the body while the sidebar stays fixed.
- The sidebar has `flex-shrink: 0` with `width: max-content`, so it sizes to its content and won't shrink.
- All three components are optional — you can use only `AppBody` for a simple content shell, or `AppSidebar` + `AppBody` for a sidebar layout without a header.
- Auto-positioning works by reading `componentId` from the child element's render function — only direct children are scanned.
- Do NOT wrap children in Fragments, conditionals, or unknown elements — `AppLayout` only auto-positions components with `componentId` matching `"AppHeader"`, `"AppSidebar"`, or `"AppBody"`. Anything else is silently ignored and won't render.
- Breadcrumbs are rendered from context. Call `useBreadcrumbs()` from any child, AppLayout handles the rest. When items is empty, breadcrumbs are hidden.
