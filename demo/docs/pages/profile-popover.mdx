# ProfilePopover

The single shared profile menu used by both [`AppHeaderProfile`](/app-header#appheaderprofile) (top-right header) and [`SidebarFooter`](/sidebar#sidebarfooter) (bottom of the sidebar). Renders an avatar/name/email header (with an optional `headerAction` icon at the far right), a dynamic list of `items` rows, an optional composable section for `children` (only a `RoleSelect` element is accepted), and a **Sign Out** row. Dividers only render between sections that actually have content — nothing renders if `items` and `children` are both omitted.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `fullName` | `string` | — | Shown next to the avatar and in the popover header |
| `email` | `string` | — | Shown under the name in the popover header |
| `profilePhoto` | `string` | — | Image URL; falls back to initials when omitted or failing to load |
| `trigger` | `ReactNode` | avatar + name/email row | Custom trigger element — overrides the default |
| `headerAction` | `ReactNode` | — | Rendered at the far right of the popover header row (e.g. a theme toggle icon button) |
| `items` | `ProfilePopoverItem[]` | — | Menu rows rendered below the header |
| `onSignOut` | `() => void` | — | Called when "Sign Out" is clicked |
| `loading` | `boolean` | `false` | Shimmer the default trigger and the header/menu rows — see [Loading state](#loading-state) |
| `loadingLabel` | `string` | `"Loading profile"` | Accessible label announced while loading |
| `children` | `ReactElement<RoleSelectProps>` | — | Extra composable content rendered between `items` and "Sign Out" — only a `RoleSelect` element is accepted |
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"top"` | Popover placement side |
| `align` | `"start" \| "center" \| "end"` | `"start"` | Popover placement alignment |
| `sideOffset` | `number` | `8` | Distance from the trigger |
| `className` | `string` | — | Merged onto the popover content element |

### `ProfilePopoverItem`

| Key | Type | Description |
|-----|------|-------------|
| `icon` | `ReactNode` | Leading icon, e.g. a lucide-react icon element |
| `label` | `ReactNode` | Row label |
| `onClick` | `() => void` | Called when the row is clicked |
| `danger` | `boolean` | Renders the row in the destructive/danger color |

## Notes

- The trigger and popover header avatars use the shared `Avatar` component's `backgroundVar="--gsl-profile-avatar-bg"` — `--gsl-profile-avatar-bg` is unset by default, so avatars show their normal per-name gradient. Set that CSS custom property on an ancestor (e.g. `:root { --gsl-profile-avatar-bg: var(--gsl-primary); }`) to force a solid color for both the sidebar footer avatar and the app header profile avatar instead.
- `children` renders flush, with no extra card/padding — the [`RoleSelect`](/role-select) dropdown reads as just another row in the same menu.
- Styling (background, border, radius, shadow, z-index, font sizes) intentionally matches the rest of the popover-family components (`Popover`, `AppSwitcher`'s panel, `Dropdown`'s menu) — see [Popover](/popover) for the shared conventions.
