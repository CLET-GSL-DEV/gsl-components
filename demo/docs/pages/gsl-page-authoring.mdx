# GSL Page Authoring Skill

An agent skill that constrains page/component creation to the specific GSL library patterns used in EVS frontend apps and the `Dashboard2` / `DemoLayout2` reference implementations.

## Purpose

This is **not** a general frontend design skill. It exists to enforce a specific, proven set of component variants, layout shell composition, and data-flow patterns so that every new page looks and works like the existing EVS dashboards and the new design system demo.

The skill lives at `~/.pi/agent/skills/gsl-page-authoring/SKILL.md` and activates when the user asks to build a new page, screen, dashboard, or layout in any GSL-based project.

## What it covers

| Area | Enforced pattern |
| ------ | ----------------- |
| Layout shell | `AppLayout variant="stacked"` + `AppHeader variant="plain"` + `Sidebar variant="plain"` + collapsible `SidebarGroup`s + `SidebarFooter` with `ProfilePopover` |
| Branding | `AppHeaderBranding` only — never duplicated in sidebar |
| Page structure | `flex flex-col gap-body` (EVS) or `SectionHeader` + metrics + table (Dashboard2) |
| Table family | `variant="panel"` + `variant="spread"` + `className="gsl-table__footer--no-border"` |
| Component variants | `MetricCard variant="outline"`, `Tabs variant="pill"`, etc. |
| Loading states | Built-in `loading` props only — never hand-rolled |
| Error states | Centered `Card` with icon + message |
| Role switcher | Same state across all three locations (header, sidebar, launchpad) |
| Performance | `useMemo`/`useCallback` on every derived value and handler |
| JSX hygiene | No inline ternaries, no magic numbers |

## Full reference

See the skill file at `~/.pi/agent/skills/gsl-page-authoring/SKILL.md` for the complete rule set with code templates for every pattern.
