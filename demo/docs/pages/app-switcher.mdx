# AppSwitcher

Google Apps–style 9-dot launcher for switching between GSL systems. Drop it into your header to let users jump between products. Pass `apps` from your own data layer and use `loading` while data is being fetched.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `apps` | `AppItem[]` | — | Apps to show in the grid (use `[]` while loading) |
| `loading` | `boolean` | `false` | Shows a spinner in the panel instead of the grid |
| `loadingLabel` | `string` | `"Loading systems..."` | Accessible label while loading |
| `columns` | `number` | `3` | Grid column count |
| `maxItems` | `number` | `6` | Maximum number of apps rendered in the grid |
| `title` | `string` | — | Panel header title |
| `onAppSelect` | `(app: AppItem) => void` | — | Called when an app is selected |
| `open` | `boolean` | — | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | — | Open state callback |
| `triggerLabel` | `string` | `"Open app switcher"` | Accessible label for the trigger |
| `trigger` | `ReactNode` | 9-dot icon | Custom trigger element |
| `footer` | `ReactNode` | — | Footer content below the grid (and below `children`, if present) |
| `children` | `ReactElement<RoleSelectProps>` | — | A [`RoleSelect`](/role-select) rendered directly below the grid, with no divider/border — only a `RoleSelect` element is accepted |
| `placement` | `"bottom-end" \| "bottom-start" \| "bottom"` | `"bottom-end"` | Panel position |
| `closeOnSelect` | `boolean` | `true` | Close panel after selection |
| `className` | `string` | — | Root CSS class |
| `style` | `CSSProperties` | — | Root inline styles |

## Types

### `AppItem`

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier |
| `name` | `string` | Display name below the icon |
| `icon` | `ReactNode \| string` | Icon node, image URL (rendered as a round image), or emoji |
| `href` | `string` | Optional navigation URL |
| `onClick` | `(app: AppItem) => void` | Optional click handler |
| `disabled` | `boolean` | Disables the app cell |
| `badge` | `string` | Optional badge text |

### `AppSwitcherProps`

Props for `AppSwitcher`. Requires `apps`; supports `loading`, `loadingLabel`, and panel/trigger customization.

Also exported: `UseAppSwitcherOptions`, `UseAppSwitcherReturn`.

## Hooks

**`useAppSwitcher`** — headless open/close state for custom UI:

```tsx title="App.tsx"
import { useAppSwitcher } from "@rfdtech/components";

const { open, toggle, close, triggerRef, panelRef } = useAppSwitcher();
```

## Design tokens

| Token | Default | Used for |
|-------|---------|----------|
| `--gsl-shadow` | `--gsl-shadow-sm` | Shadow theming hook on the root element |
| `--gsl-item-size` | `80px` | Width and height of each app grid item |
| `--gsl-icon-size` | `32px` | Width and height of the icon slot inside each item |
| `--gsl-app-switcher-font` | `--gsl-font-body` | Font family for the root element |

## Notes

- Image URLs on `AppItem.icon` render as round cropped icons inside the grid.
- Fetch apps in your application and pass `apps` + `loading`; AppSwitcher does not call APIs internally.
- Also exported: `AppSwitcherItem`, `GridIcon`, `SystemAppIcon`, `AppIconTile`.
- For systems without a logo yet, pass `icon={<SystemAppIcon name="System Name" />}` — it renders a rounded-square tile with a deterministic gradient background, a subtle pattern texture, and the system's initials, all derived from `name` via a hash (same name always produces the same tile). Build a custom glyph-on-tile icon the same way with `<AppIconTile name="System Name"><YourIcon /></AppIconTile>`.
- Override shared tokens on `.gsl-app-switcher` for theming (see [Theme](/docs/theme)).
