# Hooks

Shared React hooks exported from `@rfdtech/components`. The first hook family syncs `Dialog` and `Modal` open state with URL search params for deep links and back-button dismissal. Optional **flat data params** carry scalar context into overlays (e.g. `dialog.userId=42`). For toast notifications, see [`useToast`](/docs/toast) on the Toast docs page.

## `useDialogSearchParam`

Syncs a dialog's open state to a search param (default key `dialog`).

```tsx
import {
  Dialog,
  DialogContent,
  DialogPortal,
  DialogOverlay,
  DialogTitle,
  useDialogSearchParam,
} from "@rfdtech/components";

const { open, data, onOpenChange, openWith } = useDialogSearchParam<{
  userId: string;
}>("edit-profile");

openWith({ userId: "42" });

<Dialog open={open} onOpenChange={onOpenChange}>
  <DialogPortal>
    <DialogOverlay />
    <DialogContent>
      <DialogTitle>Edit user {data?.userId}</DialogTitle>
    </DialogContent>
  </DialogPortal>
</Dialog>
```

| Argument / option | Type | Default | Description |
| ------------------- | ------ | --------- | ------------- |
| `id` | `string \| { id: string }` | — | Overlay id stored as the param value when open |
| `param` | `string` | `"dialog"` | Query param key |
| `dataPrefix` | `string` | `` `${param}.` `` | Prefix for flat data keys (e.g. `dialog.userId`) |
| `replace` | `boolean` | `false` | Use history `replaceState` instead of `pushState` |
| `adapter` | `SearchParamAdapter` | browser adapter | Custom search-param read/write integration |

Returns `{ open, data, setOpen, onOpenChange, openWith }`. `onOpenChange` is Radix-compatible (`boolean` only). Use `openWith(data)` to open with scalar fields.

## Flat data params

When open, scalar fields are stored as prefixed query keys:

| URL | Meaning |
| ----- | --------- |
| `?dialog=edit-user` | Dialog `edit-user` is open |
| `?dialog=edit-user&dialog.userId=42` | Same dialog with `userId` context |
| `?dialog=edit-user&dialog.tab=profile` | Additional scalar field |

Supported value types on write: `string`, `number`, `boolean`. `null` / `undefined` are omitted. Parsed `data` values are always **strings** from the URL — coerce in your UI as needed. Nested objects and arrays are not supported.

- `setOpen(true)` — open id only
- `setOpen({ id, data })` — open with id and data (`id` must match the hook)
- `openWith(data)` — shorthand for opening the hook's id with data
- `onOpenChange(false)` — close and remove the id plus all `${dataPrefix}*` keys

## `useModalSearchParam`

Same API as `useDialogSearchParam` with default param key `modal`.

```tsx
const { open, data, onOpenChange, openWith } = useModalSearchParam<{
  importId: string;
}>("bulk-import");

openWith({ importId: "batch-12" });

<Modal open={open} onOpenChange={onOpenChange}>
  <ModalContent>Import {data?.importId}</ModalContent>
</Modal>
```

## `useSearchParamOverlay`

Lower-level hook for any overlay type (e.g. `Sheet` with `param: "sheet"`).

```tsx
const { open, data, openWith } = useSearchParamOverlay<{ panel: string }>(
  "settings",
  { param: "sheet" },
);
```

## React Router

Table hooks (`useTableFilter`, `useTablePagination`, `useTableState`) and components (`TableSearch`, `TableFilter`, `TablePagination`, `SidebarLink`, `AppLayoutInner`) now use a `RouterAdapter` abstraction — no manual adapter wiring needed. See the [RouterAdapter](/docs/router-adapter) docs page.

The `SearchParamAdapter` below is a separate pattern for `useSearchParamOverlay`, `useDialogSearchParam`, and `useModalSearchParam`. Pass an adapter built from `useSearchParams` so the hook stays in sync with React Router:

```tsx
import { useMemo } from "react";
import { useSearchParams } from "react-router-dom";
import {
  createSearchParamAdapter,
  useDialogSearchParam,
} from "@rfdtech/components";

function EditProfileDialog() {
  const [searchParams, setSearchParams] = useSearchParams();
  const adapter = useMemo(
    () =>
      createSearchParamAdapter({
        get: () => searchParams,
        set: (next, options) => setSearchParams(next, options),
      }),
    [searchParams, setSearchParams],
  );

  const { open, data, onOpenChange, openWith } = useDialogSearchParam<{
    userId: string;
  }>("edit-profile", { adapter });

  // ...
}
```

## Adapters

### `createBrowserSearchParamAdapter`

Default adapter using `window.location` and the History API. Subscribes to `popstate` for back/forward navigation.

### `createSearchParamAdapter`

Factory for custom adapters:

```ts
createSearchParamAdapter({
  get: () => searchParams,
  set: (next, options) => setSearchParams(next, options),
  subscribe?: (onChange) => () => void,
});
```

### Data utilities

`readOverlayData`, `writeOverlayData`, `clearOverlayData`, and `getDataPrefix` for advanced custom integrations.

## Types

### `SearchParamAdapter`

| Member | Description |
| -------- | ------------- |
| `getSearchParams` | Returns current `URLSearchParams` |
| `setSearchParams` | Writes params; optional `{ replace }` |
| `subscribe` | Optional listener for external param changes |

### `SearchParamOverlayState<TData>`

`{ id: string; data?: TData }` for `setOpen` and typed overlay context.

### `SearchParamOverlayData`

`Record<string, string | number | boolean | null | undefined>` — scalar fields only.

### `UseSearchParamOverlayReturn<TData>`

| Member | Type | Description |
| -------- | ------ | ------------- |
| `open` | `boolean` | Whether this overlay id is active |
| `data` | `Partial<Record<keyof TData, string>> \| null` | Parsed flat params when open |
| `setOpen` | `boolean \| SearchParamOverlayState<TData>` | Open, close, or open with state |
| `onOpenChange` | `(open: boolean) => void` | Radix-compatible close handler |
| `openWith` | `(data?: TData) => void` | Open this overlay id with optional data |

## Notes

- One param key holds one overlay id at a time (`?dialog=edit-profile`).
- Dialog and modal params are independent (`?dialog=settings&modal=import`).
- Closing only removes the param and data keys when the current value matches the hook's `id`.
- Without an adapter, the browser default works for non-router apps; use `createSearchParamAdapter` with your router in SPA apps.

## `useConfirmBeforeUnload`

Shows a browser confirmation prompt when the user tries to close, refresh, or navigate away from the page. Useful for modals with unsaved progress — like the `BulkImportModal`.

```tsx
import { useConfirmBeforeUnload } from "@rfdtech/components";

function MyModal({ hasUnsavedData }) {
  useConfirmBeforeUnload(hasUnsavedData);

  // ...
}
```

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `shouldConfirm` | `boolean` | — | When `true`, intercepts `beforeunload` to show the browser prompt |
| `message` | `string` | `""` | Custom message (most browsers ignore this and show their own) |

**Returns:** `void`

> The `beforeunload` event fires for page refresh, tab close, and external navigation. It does **not** fire for in-app route changes — pair it with React Router's `useBlocker` if needed.

## `useDebounce`

Debounces a value by a given delay in milliseconds. Returns the latest value after the delay has elapsed without further changes.

```tsx
import { useDebounce } from "@rfdtech/components";

const [search, setSearch] = useState("");
const debouncedSearch = useDebounce(search, 300);

// Use debouncedSearch for API calls, filter the table, etc.
```

| Argument | Type | Description |
|----------|------|-------------|
| `value` | `T` | The value to debounce |
| `delay` | `number` | Delay in milliseconds before the value updates |

**Returns:** the debounced value of type `T`.

## `useTableState`

Reads all table state (page, pageSize, search, filters, sort) from URL search params. Use to derive filtered and paged data.

```tsx
import { useTableState } from "@rfdtech/components";

const { page, pageSize, search, filters, pageSizeOptions } = useTableState({
  defaultPageSize: 10,
  paramPrefix: "members",
});

const filtered = data.filter((item) =>
  item.name.toLowerCase().includes(search.toLowerCase()),
);
const totalPages = Math.ceil(filtered.length / pageSize);
const paged = filtered.slice((page - 1) * pageSize, page * pageSize);
```

| Option | Type | Default |
| -------- | ------ | --------- |
| `defaultPage` | `number` | `1` |
| `defaultPageSize` | `number` | `10` |
| `pageSizeOptions` | `number[]` | `[10,20,50,100]` |
| `defaultSearch` | `string` | `""` |
| `defaultFilters` | `Partial<TFilters>` | — |
| `defaultSort` | `{ column, direction } \| null` | `null` |
| `paramPrefix` | `string` | — |

**Returns:** `{ page, pageSize, pageSizeOptions, setPage, setPageSize, search, setSearch, filters, setFilters, setFilter, sort, setSort, resetAll }`

See the [Table](/docs/table) docs for full component integration.

## `useTablePagination`

Standalone pagination via URL params. Use when you only need page/pageSize, without search or filters.

```tsx
const { page, pageSize, setPage, setPageSize, reset } = useTablePagination({
  defaultPageSize: 10,
});
```

| Option | Type | Default |
| -------- | ------ | --------- |
| `defaultPage` | `number` | `1` |
| `defaultPageSize` | `number` | `10` |
| `paramPrefix` | `string` | — |

**Returns:** `{ page, pageSize, setPage, setPageSize, reset }`
