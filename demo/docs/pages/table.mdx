# Table

Compound table component with built-in URL-driven search, pagination, filters, column sorting, selection, bulk actions, and loading skeletons. All state lives in the URL — components read and write search params internally. Pass `paramPrefix` to `<Table>` to namespace state for multiple tables on one page.

## Architecture

Every child component reads and writes its own URL keys through `useSearchParams`. The root `<Table paramPrefix>` provides a shared namespace via React Context. No prop drilling — pass `totalPages` and `totalItems` to `<TablePagination>`; everything else is fully automatic.

```
<Table paramPrefix="members">
  <TableSearch />          → writes ?members.search=term
  <TableFilter>            → writes ?members.f_role=admin
    <select name="role">   → FormData captured on Apply
  </TableFilter>
  <TableContent />         → renders data + sort headers
  <TableBulkActions />     → shows selection bar with inline actions
  <TablePagination />      → reads/writes ?members.page=1&members.pageSize=10
</Table>
```

Multiple tables on one page share the same URL without collision by using different prefixes:

```tsx
<Table paramPrefix="members">{/* ... */}</Table>
<Table paramPrefix="invoices">{/* ... */}</Table>
// URL: ?members.page=1&invoices.page=2&invoices.search=paid
```

## `<Table>`

Root wrapper. Accepts a `paramPrefix` that is provided via Context to all children.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `paramPrefix` | `string` | — | **Required.** URL key namespace shared by all child components |
| `classNames` | `TableClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root div |

## `<TableSearch>`

Search input with debounce and a clear button. Writes the search term to `?search=` (or `?prefix.search=`) in the URL, resets page to 1 on change.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `placeholder` | `string` | `"Search..."` | Placeholder text |
| `debounceMs` | `number` | `300` | Debounce delay in ms |
| `onSearch` | `(value: string) => void` | — | Callback that fires on the **debounced** search value (not every keystroke) |
| `onChange` | `(e: ChangeEvent) => void` | — | Raw input change event (fires on every keystroke) |
| `className` | `string` | — | Merged onto the search wrapper |

## `<TableFilter>`

Popover with filter fields and Apply / Reset buttons. Wraps children in a `<form>` element. On Apply, reads `FormData` from named fields and writes `f_{name}={value}` to the URL. On Reset, clears all `f_*` params.

Filter fields must have a `name` attribute, e.g. `<select name="status">`. The name becomes the URL key.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | Filter form content (inputs with `name` attributes) |
| `onApply` | `() => void` | — | Optional callback when Apply is clicked (URL write happens first) |
| `onReset` | `() => void` | — | Optional callback when Reset is clicked (URL clear happens first) |
| `applyLabel` | `string` | `"Apply Filter"` | Apply button text |
| `resetLabel` | `string` | `"Reset"` | Reset button aria-label |
| `className` | `string` | — | Merged onto the root div |

## `<TableContent>`

Renders a `<table>` with columns, data, optional sort, row selection, and loading skeletons.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `columns` | `TableColumn<T>[]` | — | Column definitions |
| `data` | `T[]` | — | Row data |
| `rowKey` | `(row: T) => string \| number` | — | Unique key extractor for each row |
| `loading` | `boolean` | `false` | Show skeleton rows instead of data |
| `loadingRows` | `number` | `5` | Number of skeleton rows to display |
| `selectable` | `boolean` | `false` | Show a checkbox selection column with select-all in the header |
| `selectedIds` | `Set<string \| number>` | — | Set of selected row keys. Pass with `onSelectionChange` when `selectable` is true |
| `onSelectionChange` | `(selectedIds: Set<string \| number>) => void` | — | Called with a new `Set` on any selection toggle |
| `className` | `string` | — | Merged onto the wrapper div |

When `selectable` is true, a checkbox column is prepended to the table. The header checkbox toggles all visible rows (shows indeterminate state when partially selected). Individual row checkboxes toggle that row by its `rowKey` value. Selection is fully controlled: provide `selectedIds` and `onSelectionChange` to manage state externally.

### `TableColumn<T>`

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `id` | `string` | — | Unique column identifier |
| `header` | `string` | — | Column header text |
| `accessorKey` | `keyof T` | — | Property path on the row |
| `accessorFn` | `(row: T) => ReactNode` | — | Custom value extractor (takes priority over `accessorKey`) |
| `cell` | `(info: { row: T; value: ReactNode }) => ReactNode` | — | Custom cell renderer |
| `width` | `number` | — | Fixed width in px |
| `minWidth` | `number` | `120` | Minimum width (used when `width` is omitted) |
| `maxWidth` | `number` | — | Maximum width in px |
| `sortable` | `boolean` | `false` | Enable column sorting (client-side only) |
| `align` | `"left" \| "center" \| "right"` | `"left"` | Text alignment |

## `<TableBulkActions>`

A bar that appears when rows are selected, showing "Selected N" on the left and action buttons on the right. Place it as a child of `<Table>`, between `<TableContent>` and `<TableFooter>`. Auto-hides when no rows are selected (`selectedIds.size === 0`).

Action handlers receive `selectedIds` automatically — consumers never reference the selection state directly in their handlers.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `selectedIds` | `Set<string \| number>` | — | **Required.** Currently selected row keys. Auto-hides when empty. |
| `selectedCount` | `number` | `selectedIds.size` | Override displayed count |
| `onClear` | `() => void` | — | Called when the X clear button is clicked |
| `actions` | `TableBulkAction[]` | — | Predefined actions rendered as inline buttons |
| `renderActions` | `({ selectedIds }) => ReactNode` | — | Render prop for custom action content. Replaces the default buttons. |
| `classNames` | `TableBulkActionsClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root bar |

### Basic usage

```tsx
const [selected, setSelected] = useState<Set<string | number>>(new Set());

<Table paramPrefix="users">
  <TableContent
    selectable
    selectedIds={selected}
    onSelectionChange={setSelected}
    columns={columns}
    data={users}
    rowKey={(u) => u.id}
  />
  <TableBulkActions
    selectedIds={selected}
    actions={[
      {
        id: "delete",
        label: "Delete",
        icon: <Trash2 size={14} />,
        onClick: (ids) => deleteUsers(ids),
        destructive: true,
      },
    ]}
  />
</Table>
```

### Custom render

Use `renderActions` to replace the button group with your own content:

```tsx
<TableBulkActions
  selectedIds={selected}
  renderActions={({ selectedIds }) => (
    <>
      <Button size="sm" variant="outline" onClick={() => handleCustom(selectedIds)}>
        Custom
      </Button>
    </>
  )}
/>
```

## `<TablePagination>`

Renders "Showing X-Y of Z" with a page-size dropdown on the left, and page number buttons with Previous/Next on the right. Reads `page` and `pageSize` from the URL; writes changes back. Uses a sliding window with ellipsis when total pages exceed `visiblePages`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `totalPages` | `number` | — | Total number of pages |
| `totalItems` | `number` | — | Total item count (shown as "Showing X–Y of N") |
| `pageSizeOptions` | `number[]` | `[10,20,50,100]` | Options in the page-size dropdown |
| `visiblePages` | `number` | `10` | Max page number buttons before ellipsis |
| `className` | `string` | — | Merged onto the root div |

## `useTableState`

Hook that reads all table state from URL params. Use it to derive filtered/paged data from your raw dataset.

```tsx
const { page, pageSize, search, filters, pageSizeOptions } = useTableState({
  defaultPageSize: 10,
  paramPrefix: "members",
});
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `defaultPage` | `number` | `1` | Initial page when URL has no value |
| `defaultPageSize` | `number` | `10` | Initial page size |
| `pageSizeOptions` | `number[]` | `[10,20,50,100]` | Options passed to pagination dropdown |
| `defaultSearch` | `string` | `""` | Initial search term |
| `defaultFilters` | `Partial<TFilters>` | — | Initial filter defaults |
| `defaultSort` | `{ column, direction } \| null` | `null` | Initial sort state |
| `paramPrefix` | `string` | — | Must match `<Table paramPrefix>` |

**Returns:** `{ page, pageSize, pageSizeOptions, setPage, setPageSize, search, setSearch, filters, setFilters, setFilter, sort, setSort, resetAll }`

Filters are stored as `Record<string, string>`. Values from URL are always strings — coerce in your filtering logic as needed. Use `filters.status ?? ""` to provide a default.

## `useTableFilter`

Standalone hook for URL-driven filter state without search, pagination, or sort.

```tsx
import { useTableFilter } from "@rfdtech/components";

const { filters, setFilter, setFilters, resetFilters } = useTableFilter({
  defaults: { role: "admin" },
  paramPrefix: "members",
});
```

| Option | Type | Default |
|--------|------|---------|
| `defaults` | `Record<string, string>` | — | Default filter values |
| `paramPrefix` | `string` | — | URL key namespace |

**Returns:** `{ filters, setFilter, setFilters, resetFilters }`

## `useTablePagination`

Standalone hook for URL-driven pagination without search or filters.

```tsx
const { page, pageSize, setPage, setPageSize, reset } = useTablePagination({
  defaultPageSize: 10,
});
```

| Option | Type | Default |
|--------|------|---------|
| `defaultPage` | `number` | `1` |
| `defaultPageSize` | `number` | `10` |
| `paramPrefix` | `string` | — |

**Returns:** `{ page, pageSize, setPage, setPageSize, reset }`

## Notes

- All URL writes are `replace: true` except page changes which use `pushState` for back-button support.
- Changing search, filters, or page size automatically resets to page 1.
- Sort is client-side only (`localeCompare`). No server integration.
- Loading skeletons use a pulse animation; reduced-motion users get a static version.
- Without a `paramPrefix`, keys are plain (`page`, `search`, `f_status`). With one, they are namespaced (`members.page`, `members.search`, `members.f_status`).
- `paramPrefix` is required on `<Table>` — every table must have a unique namespace to avoid URL collisions.
- The filter form captures all elements with a `name` attribute via `FormData`. For custom inputs (date pickers, multi-selects), ensure they set a `name` and participate in `FormData`.
- Selection is page-local: "select all" only selects rows on the current page. Selected state is fully controlled via `selectedIds` / `onSelectionChange`.
- The select-all checkbox shows an indeterminate dash when some (but not all) rows on the page are selected. Clicking it selects all visible rows; clicking again deselects them.
