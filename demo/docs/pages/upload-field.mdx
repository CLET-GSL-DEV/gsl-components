# UploadField

File upload with a dashed-border dropzone, cloud upload icon, auto-generated support text, file cards with type icons and size, and a primary action button. Supports single and multiple files, accept filters, max size validation, controlled/uncontrolled value, and `react-hook-form` integration.

## Usage with react-hook-form

`UploadField` integrates with react-hook-form through `FormField`. Pass `field.value` and `field.onChange` directly:

### Native validation

```tsx
import { useForm } from "react-hook-form";
import { Form, FormField, Field, FieldControl, FieldLabel, FieldError, UploadField } from "@rfdtech/components";

function UploadForm() {
  const form = useForm<{ file: File | null }>({
    defaultValues: { file: null },
  });

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit((data) => console.log(data.file))}>
        <FormField
          control={form.control}
          name="file"
          rules={{ required: "Please select a file" }}
          render={({ field, fieldState }) => (
            <Field invalid={!!fieldState.error}>
              <FieldLabel>Document</FieldLabel>
              <FieldControl>
                <UploadField
                  value={field.value}
                  onChange={field.onChange}
                />
              </FieldControl>
              <FieldError>{fieldState.error?.message}</FieldError>
            </Field>
          )}
        />
      </form>
    </Form>
  );
}
```

### Zod validation

Install the optional peer dependencies:

```bash
npm install react-hook-form zod @hookform/resolvers
```

Define a Zod schema with `z.instanceof(File)` and refine for file type and size:

```tsx
import { zodResolver } from "@hookform/resolvers/zod";
import { useForm } from "react-hook-form";
import { z } from "zod";
import {
  Button,
  Field,
  FieldControl,
  FieldError,
  FieldLabel,
  Form,
  FormField,
  UploadField,
} from "@rfdtech/components";

const ACCEPTED_TYPES = ["image/png", "image/jpeg", "application/pdf"];
const MAX_SIZE = 5 * 1024 * 1024; // 5 MB

const uploadSchema = z.object({
  file: z
    .instanceof(File, { message: "Please select a file" })
    .refine((f) => ACCEPTED_TYPES.includes(f.type), {
      message: "Only PNG, JPEG, and PDF files are accepted",
    })
    .refine((f) => f.size <= MAX_SIZE, {
      message: "File size must be 5 MB or less",
    }),
});

type UploadFormValues = z.infer<typeof uploadSchema>;

function UploadForm() {
  const form = useForm<UploadFormValues>({
    resolver: zodResolver(uploadSchema),
    defaultValues: { file: null },
  });

  const onSubmit = (data: UploadFormValues) => console.log(data.file?.name);

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)}>
        <FormField
          control={form.control}
          name="file"
          render={({ field, fieldState }) => (
            <Field invalid={!!fieldState.error}>
              <FieldLabel>Upload file</FieldLabel>
              <FieldControl>
                <UploadField
                  accept=".png,.jpg,.jpeg,.pdf"
                  maxSize={MAX_SIZE}
                  value={field.value}
                  onChange={field.onChange}
                />
              </FieldControl>
              <FieldError>{fieldState.error?.message}</FieldError>
            </Field>
          )}
        />
        <Button type="submit" variant="primary">
          Submit
        </Button>
      </form>
    </Form>
  );
}
```

Omit per-field `rules` when using a Zod schema resolver — Zod handles all validation.

For multiple files use `z.array(z.instanceof(File))`:

```tsx
const uploadSchema = z.object({
  files: z
    .array(z.instanceof(File))
    .min(1, "Select at least one file")
    .refine((files) => files.every((f) => f.size <= MAX_SIZE), {
      message: "Each file must be 5 MB or less",
    }),
});
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `accept` | `string` | — | Accepted MIME types (also drives the subtitle text) |
| `multiple` | `boolean` | `false` | Allow multiple file selection |
| `maxSize` | `number` | — | Maximum file size in bytes (drives the subtitle text; files exceeding this are silently filtered) |
| `value` | `File \| File[] \| null` | — | Controlled value |
| `onChange` | `(file: File \| File[] \| null) => void` | — | Change handler |
| `invalid` | `boolean` | `false` | Show error border styling |
| `disabled` | `boolean` | `false` | Disable interaction and dim appearance |
| `name` | `string` | — | Form field name (for react-hook-form) |
| `classNames` | `UploadFieldClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root element |

Also accepts remaining `HTMLAttributes<HTMLDivElement>`, excluding `onChange`, `value`, and `defaultValue`.

## File type detection

When files are added, the component detects the file type and shows a matching icon:

| Type | Detection | Icon |
|------|-----------|------|
| Image | `type.startsWith("image/")` or extension `.png`, `.jpg`, `.webp`, etc. | `FileImage` |
| Video | `type.startsWith("video/")` or extension `.mp4`, `.webm`, etc. | `FilePlay` |
| PDF | `.pdf` extension | Custom PDF icon |
| Other | Everything else | `FileText` |

File size is formatted automatically: `B`, `KB`, or `MB`.

## Notes

- The subtitle is auto-generated from `accept` and `maxSize` props (e.g. "Only CSV files are supported. Maximum filesize 10MB.").
- The action button changes text: "Upload file" when empty, "Replace file" when a file is selected. Only shown in single-file mode (`multiple={false}`).
- Drop zone turns `--gsl-primary-light` background when files are selected or during drag-over.
- When `multiple` is true, selecting or dropping new files **replaces** the current set (no append).
- Controlled mode: pass `value` and `onChange`. Uncontrolled mode: let the component manage its own state.
- Files exceeding `maxSize` are silently filtered out. No error message is shown — handle validation externally via `onChange`.
- Delete button color uses `--gsl-error` on hover.
