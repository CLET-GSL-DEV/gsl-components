# UploadField

File upload with drag-and-drop zone, file type icons (PDF, image, video, generic), file size display, and remove button. Supports single and multiple files, accept filters, max size validation, controlled/uncontrolled value, and `react-hook-form` integration.

## Usage with react-hook-form

`UploadField` integrates with react-hook-form through `FormField`. Pass `field.value` and `field.onChange` directly:

```tsx
import { useForm } from "react-hook-form";
import { Form, FormField, Field, FieldControl, FieldLabel, FieldError, UploadField } from "@rfdtech/components";

function UploadForm() {
  const form = useForm<{ file: File | null }>({
    defaultValues: { file: null },
  });

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit((data) => console.log(data.file))}>
        <FormField
          control={form.control}
          name="file"
          rules={{ required: "Please select a file" }}
          render={({ field, fieldState }) => (
            <Field invalid={!!fieldState.error}>
              <FieldLabel>Document</FieldLabel>
              <FieldControl>
                <UploadField
                  value={field.value}
                  onChange={field.onChange}
                />
              </FieldControl>
              <FieldError>{fieldState.error?.message}</FieldError>
            </Field>
          )}
        />
      </form>
    </Form>
  );
}
```

`UploadField` also works outside of `FormField` — just pass `onChange` as the controlled callback and `value` as the current file.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `accept` | `string` | — | Accepted MIME types (passed to the file input) |
| `multiple` | `boolean` | `false` | Allow multiple file selection |
| `maxSize` | `number` | — | Maximum file size in bytes (files exceeding this are silently filtered) |
| `value` | `File \| File[] \| null` | — | Controlled value |
| `onChange` | `(file: File \| File[] \| null) => void` | — | Change handler |
| `invalid` | `boolean` | `false` | Show error border styling |
| `disabled` | `boolean` | `false` | Disable interaction and dim appearance |
| `placeholder` | `string` | `"Drop files here or click to browse"` | Placeholder text in the drop zone |
| `name` | `string` | — | Form field name (for react-hook-form) |
| `classNames` | `UploadFieldClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root element |

Also accepts remaining `HTMLAttributes<HTMLDivElement>`, excluding `onChange` and `value`.

## File type detection

When files are added, the component detects the file type and shows a matching icon:

| Type | Detection | Icon |
|------|-----------|------|
| Image | `type.startsWith("image/")` or extension `.png`, `.jpg`, `.webp`, etc. | `FileImage` |
| Video | `type.startsWith("video/")` or extension `.mp4`, `.webm`, etc. | `FilePlay` |
| PDF | `.pdf` extension | Custom PDF icon |
| Other | Everything else | `FileText` |

File size is formatted automatically: `B`, `KB`, or `MB` as appropriate.

## Notes

- Drop zone is keyboard accessible — focus with Tab, activate with Enter or Space.
- Disabled state dims to `opacity: 0.6` with `cursor: not-allowed` and prevents click/drop.
- Invalid state applies `--gsl-error` border and a `1px` error box-shadow.
- When `multiple` is true, selecting or dropping a new set of files **replaces** the current set (no append).
- Controlled mode: pass `value` and `onChange`. Uncontrolled mode: let the component manage its own state.
- Files exceeding `maxSize` are silently filtered out. No error message is shown — handle validation externally via `onChange`.
