# DateRangeSelector

Date range picker for choosing a start and end date from a two-month calendar popover. Built on `@radix-ui/react-popover` with CLET styling, auto-swap to keep start before end, year/month dropdown selectors for quick navigation, Apply/Cancel buttons for explicit confirmation, and part-level `classNames`.

## With quick-select presets

A left rail of quick-select presets (`presets` prop) and a "Range: ..." summary in the footer — both are opt-in and only render when `presets` is passed. Selecting a preset previews it on the calendar; **Apply is still required to commit it**, same as a manual day selection. `getRange` is evaluated lazily on click, so relative presets like "Today" and "Last 7 days" stay accurate.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `DateRangeValue` | — | Controlled range (`{ start, end }`) |
| `defaultValue` | `DateRangeValue` | — | Default range when uncontrolled |
| `onChange` | `(range: DateRangeValue) => void` | — | Called when the range is confirmed via Apply |
| `placeholder` | `string` | `"Select date range"` | Shown in the trigger when no dates are selected |
| `formatOptions` | `Intl.DateTimeFormatOptions` | `{ year: "numeric", month: "short", day: "numeric" }` | Controls how dates are formatted in the trigger display |
| `invalid` | `boolean` | `false` | Show error border styling |
| `disabled` | `boolean` | `false` | Disable interaction |
| `min` | `Date` | — | Earliest selectable date |
| `max` | `Date` | — | Latest selectable date |
| `presets` | `DateRangePreset[]` | — | Left-rail quick-select presets; also enables the "Range: ..." footer summary |
| `classNames` | `DateRangeSelectorClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root wrapper |

Also accepts remaining `HTMLAttributes<HTMLDivElement>`, excluding `onChange`, `value`, and `defaultValue`.

## Usage with react-hook-form

```tsx
import { useForm } from "react-hook-form";
import { Form, FormField, Field, FieldControl, FieldLabel, FieldError, DateRangeSelector } from "@rfdtech/components";
import type { DateRangeValue } from "@rfdtech/components";

function FilterForm() {
  const form = useForm<{ dateRange: DateRangeValue }>({
    defaultValues: { dateRange: { start: null, end: null } },
  });

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit((d) => console.log(d.dateRange))}>
        <FormField
          control={form.control}
          name="dateRange"
          render={({ field, fieldState }) => (
            <Field invalid={!!fieldState.error}>
              <FieldLabel>Date range</FieldLabel>
              <FieldControl>
                <DateRangeSelector value={field.value} onChange={field.onChange} />
              </FieldControl>
              <FieldError>{fieldState.error?.message}</FieldError>
            </Field>
          )}
        />
      </form>
    </Form>
  );
}
```

## Types

### `DateRangeValue`

Value shape for `DateRangeSelector`. Holds the selected start and end dates, each `null` when unset. Start is always the earlier date.

| Key | Type | Description |
|-----|------|-------------|
| `start` | `Date \| null` | Earlier date in the range (always the smaller date) |
| `end` | `Date \| null` | Later date in the range (always the larger date) |

### `DateRangeSelectorProps`

See the props table above.

### `DateRangePreset`

One entry in the `presets` array:

| Key | Type | Description |
|-----|------|-------------|
| `label` | `string` | Rail label, e.g. `"Today"`, `"Last 7 days"`, `"All time"` |
| `getRange` | `() => DateRangeValue` | Computed on click — evaluated lazily so "Today"-relative presets stay accurate. Returning `{ start: null, end: null }` (e.g. for "All time") clears the range on Apply |

## Notes

- Click the trigger to open a two-month calendar popover. Year and month dropdowns at the top allow quick navigation; previous/next arrows shift by one month.
- First click picks the start date; a second click picks the end date. **No dates are committed until Apply is clicked.**
- Click Cancel or press Escape to discard the pending selection and restore the previous range.
- **Start is always the earlier date**—if the second pick is before the first, they swap automatically so `start < end`.
- Selecting the same day for both picks produces a single-day range where `start === end`.
- Clicking a new date when a range is already set clears the pending range and starts a new selection.
- Days strictly between start and end are highlighted with a subtle background in the calendar.
- The Apply button is disabled until at least a start date is selected, **or** a preset has been clicked (even a preset like "All time" that resolves to `{ start: null, end: null }`).
- Clicking a preset previews it on the calendar and marks that preset active; manually clicking a day afterward clears the active-preset highlight.
- On mobile (below 768px), the two months stack vertically, and the presets rail (when present) moves above the calendar as a horizontal, wrapping row.

## Don't

- Don't override the trigger's border with class overrides like `classNames={{ trigger: "!border-0 !shadow-none focus:!shadow-none" }}` — let it keep its default border, same as `DateSelector` and `Dropdown`.
