# Combobox

Text-input combobox with debounced async search. Use when options are loaded from an API based on what the user types.

## Example

<ExampleTabs title="User search" preview={<ComboboxPreview />}>

```tsx
import { Combobox } from "@rfdtech/components";

<Combobox
  ariaLabel="User"
  value={userId}
  onChange={setUserId}
  loadOptions={(query) => fetchUsers(query)}
  placeholder="Search users..."
  clearable
  getOptionLabel={(id) => cachedUsers.get(id)}
/>
```

</ExampleTabs>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `ariaLabel` | `string` | — | Accessible label for the input |
| `value` | `string \| null` | — | Selected option value |
| `onChange` | `(value: string \| null) => void` | — | Selection callback |
| `loadOptions` | `(query: string) => Promise<DropdownOption[]>` | — | Async search function |
| `placeholder` | `string` | `Search...` | Input placeholder |
| `debounceMs` | `number` | `300` | Debounce delay before searching |
| `minSearchLength` | `number` | `0` | Minimum query length before searching |
| `getOptionLabel` | `(value: string) => string \| undefined` | — | Resolve label for a value not in current results |
| `clearable` | `boolean` | `false` | Show clear button when a value is selected |

## Notes

- Shows a spinner while search is in flight.
- Pass `getOptionLabel` when the value may be set before options are loaded.
