# Notice

Boxed, persistent callout — unlike `Toast`, which is transient and stacked in a viewport, `Notice` renders inline wherever it's placed and stays until removed from the tree. Use it for scoped context ("your scope for this composer"), policy explanations, or a standing status message.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `NoticeVariant` | `"default"` | Token-driven accent + tinted background |
| `color` | `CSSProperties["color"]` | — | Custom accent color, overriding the variant's token color |
| `title` | `ReactNode` | — | Bold heading line |
| `icon` | `ReactNode` | — | Optional leading icon or dot, tinted with the accent color |
| `leftBorder` | `boolean` | `false` | Thick colored left border accent |
| `dashed` | `boolean` | `false` | Dashed outer border instead of solid |
| `classNames` | `NoticeClassNames` | — | Part-level class overrides |
| `className` | `string` | — | Merged onto the root (same as `classNames.root`) |
| `children` | `ReactNode` | — | Body content — plain text or a list |

Also accepts standard native `div` attributes (`id`, `aria-*`, `role`, etc.) — `role` defaults to `"alert"` for the `error` variant and `"status"` otherwise, but can be overridden.

## Types

### `NoticeProps`

Props for `Notice`. Extends `HTMLAttributes<HTMLDivElement>` with `title` omitted (redefined as `ReactNode` for the heading, instead of the native string-only tooltip attribute). `forwardRef`-enabled.

### `NoticeClassNames`

Part-level class names merged after base `gsl-notice*` styles:

| Key | Applied to |
|-----|------------|
| `root` | Outer `<div>` |
| `icon` | Leading icon wrapper |
| `header` | Header row wrapping `title` |
| `title` | Title `<span>` |
| `body` | Body `<div>` wrapping `children` |

### `NoticeVariant`

`"default" | "info" | "success" | "warning" | "error"`

## Design tokens

| Token | Default | Used for |
|-------|---------|----------|
| `--gsl-notice-accent` | `var(--gsl-text-secondary)` | Title/icon/left-border color; overridden per variant and by the `color` prop |
| `--gsl-notice-bg` | tinted from `--gsl-border` | Background tint; overridden per variant via `color-mix()` |
| `--gsl-notice-radius` | `var(--gsl-radius-base)` | Corner radius |
| `--gsl-notice-padding` | `16px` | Inner padding |
| `--gsl-notice-font` | `var(--gsl-font-body)` | Font family |
| `--gsl-notice-title-size` | `var(--gsl-text-size-base)` | Title font size |
| `--gsl-notice-body-size` | `var(--gsl-text-size-sm)` | Body font size |

## Notes

- No new brand colors were introduced — `info`/`success`/`warning`/`error` reuse the existing `--gsl-primary`/`--gsl-success`/`--gsl-warning`/`--gsl-error` tokens, tinted for the background via `color-mix()` (the same technique `Badge` uses).
- The `color` prop only overrides `--gsl-notice-accent` (title/icon/left-border); the background tint is not recomputed from a custom color, since `color-mix()` requires a token reference, not an arbitrary runtime value — pass a `classNames.root` override if a fully custom background is needed.
- `leftBorder` and `dashed` compose — when both are set, the left edge stays solid (using the accent color) while the remaining three edges are dashed.
