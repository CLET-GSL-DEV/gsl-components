# Migration guide: v1 → v2 — Agent Instructions

Agent-only. All breaking changes and deprecations between `1.22.0` and current, organized as directives. See [Changelog](/docs/changelog) for dated list.

## 1. `AppHeaderProfile` → `ProfilePopover`

**Deprecation, not breaking.** `AppHeaderProfile` still works (thin wrapper around `ProfilePopover`). No forced migration, no removal date. `ProfilePopover` gets all new features.

**When you encounter `AppHeaderProfile`:**

- Replace with `ProfilePopover` using `user` and `variant` props for same trigger
- `onProfileClick`/`onSettingsClick`/`onHelpClick` have NO `ProfilePopover` equivalent — build those three rows via `items` array
- `headerAction` prop silently ignored on both — delete it. Popover header now always shows built-in theme toggle when `ThemeProvider` is in tree

```diff
-import { AppHeaderProfile } from "@rfdtech/components";
+import { ProfilePopover } from "@rfdtech/components";

-<AppHeaderProfile user={user} variant="avatar">
-  <RoleSelect roles={roles} selectedRole={role} onClickRole={setRole} />
-</AppHeaderProfile>
+<ProfilePopover user={user} variant="avatar">
+  <RoleSelect roles={roles} selectedRole={role} onClickRole={setRole} />
+</ProfilePopover>
```

`items`, `loading`/`loadingLabel`, `onSignOut`, `children` carry over unchanged.

**Building menu items from `onProfileClick`/`onSettingsClick`/`onHelpClick`:**

```tsx
<ProfilePopover
  user={user}
  variant="avatar"
  items={[
    { icon: <User size={20} />, label: "My Profile", onClick: goToProfile },
    { icon: <Settings size={20} />, label: "Account Settings", onClick: goToSettings },
    { icon: <HelpCircle size={20} />, label: "Help & Support", onClick: goToHelp },
  ]}
  onSignOut={signOut}
/>
```

Only `"full"` and `"avatar"` variants remain on both components (no `"basic"`).

## 2. `AppItem.icon` → `SystemAppIcon` or `AppIconTile`

`AppSwitcher`'s `AppItem.icon` now accepts `ReactNode` only — closed to `<SystemAppIcon name="..." />` (auto-generated initials tile) or `<AppIconTile name="...">custom glyph</AppIconTile>`. Every app icon renders on the shared 9-gradient/9-overlay tile system.

```diff
-  { id: "analytics", name: "Analytics", icon: "https://cdn.example.com/analytics.png" },
+  { id: "analytics", name: "Analytics", icon: <SystemAppIcon name="Analytics" /> },
-  { id: "custom", name: "Custom App", icon: <MyCustomLogo /> },
+  { id: "custom", name: "Custom App", icon: <AppIconTile name="Custom App"><MyCustomLogo /></AppIconTile> },
];
```

## 3. `AppSwitcher` maxItems default is 6

Default `maxItems` is 6. Migrate grids with >6 apps:

```diff
-<AppSwitcher apps={apps} />
+<AppSwitcher apps={apps} maxItems={apps.length} />
```

Better: switch to `Launchpad` (see §6).

## 4. `Table`: selection-only loses kebab column

`Table` with `selectable` but no `rowActions` no longer renders an empty kebab column. If you relied on the Select/Deselect entry there, add explicit row action:

```diff
 <TableContent
   selectable
   selectedIds={selected}
   onSelectionChange={setSelected}
+  rowActions={[
+    { id: "select", label: isSelected ? "Deselect" : "Select", onClick: toggleRow },
+  ]}
 />
```

## 5. Rebrand: new default colors and typography

**If no `--gsl-*` overrides set:**

- `--gsl-primary`/`--gsl-primary-light`: red (`#dc2626`/`#ef4444`) → navy (`#051b2c`/`#4a7fa8`)
- New `--gsl-secondary` (gold `#c8a24b`) — default `SidebarLink` active-link color
- New typography: `--gsl-font-header` (Roboto Serif), `--gsl-font-body` (Lato) via `@fontsource/lato`, `@fontsource/roboto-serif`

**To override:**

```tsx
import { gslTheme } from "@rfdtech/components";
gslTheme({ all: { primary: "#dc2626", primaryLight: "#fef2f2" } });
```

```css
:root { --gsl-font-header: var(--gsl-font); --gsl-font-body: var(--gsl-font); }
```

**To keep new defaults: do nothing.** See `.legacy-theme` in `demo/demo.css` for the old red look reference.

## 6. Prefer `Launchpad` over deprecated `AppSwitcher`

`AppSwitcher` deprecated, no new features. `Launchpad` is the replacement — independent component tree.

**Not a drop-in replacement.** Re-author the app list and trigger:

```diff
-import { AppSwitcher, SystemAppIcon } from "@rfdtech/components";
+import { Launchpad, SystemLaunchpadIcon } from "@rfdtech/components";

 const apps = [
-  { id: "analytics", name: "Analytics", icon: <SystemAppIcon name="Analytics" /> },
+  { id: "analytics", name: "Analytics", icon: <SystemLaunchpadIcon name="Analytics" /> },
 ];

-<AppSwitcher apps={apps} maxItems={apps.length} footer={<CustomFooter />} />
+<Launchpad apps={apps} onAppSelect={(app) => navigate(app.href)}>
+  <RoleSelect roles={roles} selectedRole={role} onClickRole={setRole} />
+</Launchpad>
```

**Launchpad differences:**

- Icon type: `SystemLaunchpadIcon`/`LaunchpadIconTile` only (not `SystemAppIcon`/`AppIconTile`)
- Grid: fixed 3×3 (9 apps), not configurable. "See all" opens 5-column modal with every app
- No `maxItems`/`columns`/generic `footer` slot. Only `children` accepts single `RoleSelect`
- No `title`/`triggerLabel`/`loadingLabel`/`onSeeAll`/`seeAllLabel`/`placement`/`closeOnSelect`
- Title always "Launchpad", trigger always "Open Launchpad", selecting app always closes popover

**Stay on `AppSwitcher` if you need: flexible column count, uncapped grid without "See all" modal, or custom `footer`. Check with team before building new custom-footer `AppSwitcher` usage.**

## Adopting the new design system

This is the actual "v2" migration. §1–§6 are about not breaking on version bump. This section is about looking like the new design system. None required for existing screens to compile. All required for new/touched screens.

### Layout shell: `AppLayout variant="stacked"` + `AppHeader variant="plain"` + `Sidebar variant="plain"`

Adopt as a set, not individually:

```diff
-<AppLayout>
+<AppLayout variant="stacked">
   <AppHeader variant="plain">...</AppHeader>
   <AppSidebar>
     <SidebarOverlay />
-    <Sidebar>
+    <Sidebar variant="plain">
       ...
     </Sidebar>
   </AppSidebar>
   <AppBody><Outlet /></AppBody>
 </AppLayout>
```

- **`AppLayout variant="stacked"`**: header spans full width on top, sidebar + content side by side below. Default (`"default"`) runs sidebar full-height on left, header only spans content column.
- **`AppHeader variant="plain"`**: square corners, `--gsl-primary` background, on-primary text. Continuous top bar with branding.
- **`Sidebar variant="plain"`**: transparent bg + right border instead of panel surface. Removes boxed-in panel-inside-panel look.

**`SidebarGroup collapsible`:** All nav groups use accordion sections, not flat lists. Initialize with `Main` group expanded, others collapsed.

**`SidebarFooter` must host a `ProfilePopover`:** Use `fullName`/`email` mode (not `user`/`variant`). Full name, email, role get stable real-estate-rich home here. Never omit sidebar footer.

**Branding lives in `AppHeader` via `AppHeaderBranding`, NOT in sidebar:**

```tsx
<AppHeader variant="plain">
  <AppHeaderBranding
    logo={<img src="/logo.png" alt="" width={28} height={28} />}
    title="YOUR PORTAL"
    subtitle="Your Product Name"
  />
  <AppHeaderActions>...</AppHeaderActions>
</AppHeader>
```

`Sidebar` has NO `SidebarHeader`/`SidebarBrand` when `AppHeaderBranding` is used. Same logo must not appear twice on screen.

### Use built-in `loading` states — never hand-roll placeholders

Every component below supports a real `loading` prop with shimmering skeleton. Use it. Never render `"..."` placeholders, hand-rolled `<Spinner />`, or omit-until-ready.

| Component | Loading prop | Replaces |
| ----------- | ------------- | ---------- |
| `Launchpad` / `AppSwitcher` | `loading` | Disabled/empty trigger or nothing until apps arrive |
| `AppHeaderNotifications` | `loading` | Empty popover or hand-rolled spinner |
| `ProfilePopover` | `loading` + `loadingLabel` | `"..."` or nothing for name/email/avatar |
| `SidebarLink` | `loading` + `loadingLabel` | Not rendering nav items until data resolves, or full-page spinner |
| `MetricCard` | `loading` + `loadingLabel` | `"..."` or `"N/A"` in `value`/`description` |
| `TableContent` | `loading` + `loadingRows` | Empty table, spinner overlay, or hide-until-ready |

**SidebarLink pattern:**

```tsx
{navLoading ? (
  <SidebarGroup>
    {Array.from({ length: 6 }).map((_, i) => (
      <SidebarLink key={i} loading icon={<span />}>Loading</SidebarLink>
    ))}
  </SidebarGroup>
) : (
  navGroups.map((group) => /* real nav */)
)}
```

### One role switcher, consistent across all three locations

`RoleSelect` can be `children` in: `ProfilePopover` (header), `ProfilePopover` (sidebar footer), and `Launchpad`. Wire ALL THREE from the same `roles`/`selectedRole`/`onClickRole` state:

```tsx
const [selectedRole, setSelectedRole] = useState(roles[0].id);
const roleSelectProps = { title: "View as", roles, selectedRole, onClickRole: (r) => setSelectedRole(r.id) };

// Header
<ProfilePopover variant="avatar" user={user} items={items} onSignOut={onSignOut}>
  <RoleSelect {...roleSelectProps} />
</ProfilePopover>

// Sidebar footer
<SidebarFooter>
  <ProfilePopover fullName={user.name} email={user.email} items={items} onSignOut={onSignOut}>
    <RoleSelect {...roleSelectProps} />
  </ProfilePopover>
</SidebarFooter>

// Launchpad
<Launchpad apps={apps} onAppSelect={onAppSelect}>
  <RoleSelect {...roleSelectProps} />
</Launchpad>
```

**Header `ProfilePopover` uses `variant="avatar"` when sidebar footer popover exists.** Reserve `variant="full"` for headers with NO sidebar footer.

### Table family — adopt together

| Component | Prefer | Over |
| ----------- | -------- | ------ |
| `TableContent` | `variant="panel"` | `"default"` |
| `TableFilter` | `variant="spread"` | `"popover"` (default) |
| `TableFooter` | `className="gsl-table__footer--no-border"` | default top border |

```diff
 <Table paramPrefix="members">
   <TableHeader>
     <TableSearch placeholder="Search members..." />
-    <TableFilter>
+    <TableFilter variant="spread">
       <Dropdown name="role" ... />
     </TableFilter>
   </TableHeader>
-  <TableContent variant="panel" columns={columns} data={data} rowKey={...} />
+  <TableContent variant="panel" columns={columns} data={data} rowKey={...} />
-  <TableFooter>
+  <TableFooter className="gsl-table__footer--no-border">
     <TablePagination ... />
   </TableFooter>
 </Table>
```

`gsl-table__footer--no-border` is a real library class (`.gsl-table__footer--no-border { border-top: none; }`), not a demo hack.

### Other preferred variants

| Component | Prefer | Over |
|-----------|--------|------|
| `MetricCard` | `variant="outline"` | `"default"` |
| `Tabs` | `variant="pill"` | `"default"` |

Use `SectionHeader` (with `SectionTitle`/`SectionDescription`/`SectionActions`) for page headers — never hand-rolled `<div>`s.

All variants above are additive — no deadline to migrate old screens. But any screen built fresh or substantially touched must use these variants.

## Checklist

**Compatibility (keeps existing screens compiling — do when touching file):**

- [ ] Migrate `AppHeaderProfile` → `ProfilePopover` where convenient (no removal date) (§1)
- [ ] Add `RoleSelect` to `Launchpad` when migrating it
- [ ] Rebuild "My Profile"/"Account Settings"/"Help & Support" via `items` array on migration
- [ ] Convert every `AppSwitcher` → `Launchpad` when possible (§6)
- [ ] Pass `maxItems={apps.length}` to any `AppSwitcher` with >6 apps, or switch to `Launchpad` (§3)
- [ ] Add explicit `rowActions` Select/Deselect to `selectable`-only tables that need it (§4)
- [ ] Decide to keep or override new brand colors/typography (§5)

**Design system adoption (the actual "looks like v2" bar — for new/touched screens):**

- [ ] New `AppSwitcher`-like work uses `Launchpad` first (§6)
- [ ] New/rebuilt layout shells use `AppLayout variant="stacked"` + `AppHeader variant="plain"` + `Sidebar variant="plain"` + collapsible `SidebarGroup`s together ("Layout shell")
- [ ] Every layout has `SidebarFooter` with `ProfilePopover` in `fullName`/`email` mode ("Layout shell")
- [ ] Branding in `AppHeader` via `AppHeaderBranding`, never duplicated in sidebar ("Layout shell")
- [ ] Every loading-props table component uses its real `loading` prop — never hand-rolled `"..."` or spinner ("Use built-in loading states")
- [ ] Same `RoleSelect` state backs all three locations if role-switching exists ("One role switcher")
- [ ] Header `ProfilePopover` uses `variant="avatar"` when sidebar footer exists ("One role switcher")
- [ ] New/touched `Table`s use all three: `TableContent variant="panel"`, `TableFilter variant="spread"`, `TableFooter className="gsl-table__footer--no-border"`
- [ ] `MetricCard variant="outline"`, `Tabs variant="pill"` on new usage
- [ ] New page headers use `SectionHeader`/`SectionTitle`/`SectionDescription`/`SectionActions`
